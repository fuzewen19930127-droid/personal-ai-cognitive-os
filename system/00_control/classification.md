# Task Classification

Classify tasks before deciding what to read and what to update.

| Task type | Use when | Recall mode |
|---|---|---|
| `simple_qa` | One-off explanation or small answer | lightweight |
| `project_handoff` | Continuing a long-running project | standard |
| `code_implementation` | Editing code, scripts, docs, or generated assets | standard |
| `automation_design` | Designing workflows, scripts, schedulers, or data flows | enhanced |
| `content_production` | Creating posts, docs, launch assets, or portfolio material | standard |
| `technical_selection` | Comparing tools, APIs, frameworks, or architectures | enhanced |
| `learning_research` | Studying a topic or building a knowledge pack | enhanced |
| `troubleshooting` | Debugging errors or unstable behavior | standard |
| `system_governance` | Updating rules, memory, indexes, or audit policy | standard |
| `high_change_fact_check` | Current prices, laws, policies, API versions, or security advice | high_change |

## Recall Modes

| Mode | Read scope |
|---|---|
| `lightweight` | Current context only |
| `standard` | Control rules, project memory, task memory, error routes, project cards |
| `enhanced` | Standard mode plus local external knowledge indexes and digests |
| `high_change` | Enhanced mode plus current official or primary-source verification |
