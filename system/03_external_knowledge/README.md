# Local External Knowledge

This layer stores external knowledge that has been collected locally.

It can include official docs, GitHub repos, papers, tutorials, standards, engineering articles, digests, and indexes.

External knowledge is evidence and background. It does not automatically override local project rules.

## Suggested Files

| File | Purpose |
|---|---|
| `sources_manifest.md` | Source list and metadata |
| `source_index.md` | Titles, keywords, and routing notes |
| `digest.md` | Distilled reading notes |
| `candidate_rules.md` | Possible rule upgrades |

## Relationship To Memory Layers

This directory is L2.

- L1 stores project memory and rules.
- L2 stores collected external knowledge.
- L3 verifies current high-change facts.

## Source Priority

- Official docs and official repos first.
- Primary papers, standards, protocols, and source code first.
- Engineering articles can support decisions but should not set policy alone.
- Community experience is a clue, not a final source.
- APIs, prices, laws, platform rules, and versions need current verification.
